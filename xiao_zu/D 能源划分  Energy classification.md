**时间限制： 1.0 秒**

**【题目描述】**

某新能源公司运营着一条由 $n$ 个光伏阵列组成的发电带。第 $i$ 个光伏阵列在某一时刻的实际出力（单位：kW）为 $a_i$。为了接入电网，公司需要将这 $n$ 个阵列依次划分成恰好 $m$ 个连续的运行周期，每个周期至少包含一个阵列。对于每个运行周期，定义其波动风险值为：

**周期长度 ×（周期内的最大出力 - 周期内的最小出力）**

整个调度方案的总风险值定义为所有周期中波动风险值的最大值。请你设计一个划分方案，使得总风险值最小，并输出该最小值。

**【输入格式】**

从标准输入读入数据。

第一行：两个整数 $n, m$。

第二行：$n$ 个整数 $a_1, a_2, \dots, a_n$，表示每个光伏阵列的实时出力。

**【输出格式】**

输出到标准输出。

输出一个整数，表示最小可能的总风险值。

**【数据范围与约定】**

对于 40% 的数据，$1 \le m \le n \le 1000$，$1 \le a_i \le 1000$。

对于 100% 的数据，$1 \le m \le n \le 10^5$，$1 \le a_i \le 10^9$。

## **解题思路：**

本题的核心诉求是“**使得总风险值最小**”，而“总风险值”又被定义为所有划分周期中“**风险值的最大值**”。

在算法竞赛中，但凡遇到“求最大值中的最小值（最小化最大值）”这类字眼，二分答案（Binary Search on Answer）几乎是唯一且最高效的解题范式。

既然直接划分数组很难，我们不妨反向思考：

假设我们随便猜一个目标风险值 $X$，我们有没有办法验证“能否在保证每个周期风险值 $\le X$ 的前提下，将数组划分为不超过 $m$ 个周期？”

答案是可以的，并且可以用**贪心算法**在 $O(n)$ 的时间内快速验证。

**贪心验证策略**：

对于一个固定的猜测值 $X$，我们从左到右遍历光伏阵列：

把阵列逐个加入当前的周期中，并实时更新当前周期的 `Max`（最大出力）和 `Min`（最小出力），计算 `长度 × (Max - Min)`。

- 只要这个值 $\le X$，我们就继续往里加（贪心地让当前周期尽可能长，这样剩下的周期数量就会越少）。
    
- 一旦加入某个阵列后导致风险值 $> X$，说明当前周期“装不下”了，我们必须在它之前“切断”，然后新开一个周期，继续往后遍历。
    

遍历结束后，如果切分出的总周期数 $\le m$，说明我们猜的 $X$ 是可行的（甚至可能猜大了）；如果周期数 $> m$，说明 $X$ 猜小了，要求太苛刻导致碎片化严重，必须放宽条件。

## **具体解法：**

1. **确定二分边界**：
    
    - **下界 `low = 0`**：最好的情况是所有出力完全相等，最大值减最小值等于 0。
        
    - **上界 `high` 的评估**：理论最坏情况下，所有 $n$ 个阵列全部分在一个周期里（当 $m=1$ 时），极限最大风险值为 $n \times (\text{全局最大值} - \text{全局最小值})$。题目中 $n \le 10^5$，数值 $\le 10^9$，因此最大风险上限为 $10^{14}$，这个数值刚好在 64 位整型 `long long` 的安全范围内。
        
2. **实现 `check(limit)` 函数**：
    
    严格按照上述的贪心逻辑进行 $O(n)$ 遍历扫描，动态维护当前片段的极值。
    
3. **二分主框架**：
    
    使用标准的 `while (low <= high)` 二分模板不断折半逼近答案。

## **运行结果分析：**

对于样例输入，程序通过二分搜索与贪心策略，找到了最优的划分为：

- 周期1：`[4, 10]`，长度2，最大10，最小4，风险 $(10-4) \times 2 = 12$
    
- 周期2：`[1, 6]`，长度2，最大6，最小1，风险 $(6-1) \times 2 = 10$
    
- 周期3：`[9, 6]`，长度2，最大9，最小6，风险 $(9-6) \times 2 = 6$
    

总风险值为 $\max(12, 10, 6) = 12$，与程序输出完美契合。算法的时间复杂度为 $O(n \log(\text{Max\_Risk}))$。因为上限为 $10^{14}$，二分最多只会执行约 47 次，结合 $O(n)$ 的校验，总计执行不足 $5 \times 10^6$ 次运算。耗时通常只需几毫秒，轻松通过 1.0 秒的时间限制。
源代码
``` C++
#include <iostream>

#include <vector>

#include <algorithm>

  

using namespace std;

  

int n, m;

vector<long long> a;

  

// 核心验证函数：判断给定的风险阈值 limit 是否可行

bool check(long long limit) {

    int count = 1; // 至少需要 1 个周期

    long long cur_min = a[0];

    long long cur_max = a[0];

    long long len = 1;

  

    for (int i = 1; i < n; ++i) {

        // 模拟将当前元素 a[i] 加入当前周期的状态

        long long next_min = min(cur_min, a[i]);

        long long next_max = max(cur_max, a[i]);

        long long next_len = len + 1;

  

        // 如果加入后的风险值超过了限制，必须另起一个新周期

        if (next_len * (next_max - next_min) > limit) {

            count++;

            // 新周期以当前元素 a[i] 为起点重新计算

            cur_min = a[i];

            cur_max = a[i];

            len = 1;

        } else {

            // 否则，贪心地将 a[i] 纳入当前周期

            cur_min = next_min;

            cur_max = next_max;

            len = next_len;

        }

    }

    // 如果切分的周期数 <= m，说明该限制可行

    return count <= m;

}

  

int main() {

    // 优化终端 I/O 速度

    ios_base::sync_with_stdio(false);

    cin.tie(NULL);

  

    if (cin >> n >> m) {

        a.resize(n);

        for (int i = 0; i < n; ++i) {

            cin >> a[i];

        }

  

        // 设定二分的上下界

        long long low = 0;

        // 最坏情况：n = 10^5, 数据差值为 10^9，风险极值为 10^14

        long long high = 1e14;

        long long ans = high;

  

        // 二分答案模板

        while (low <= high) {

            long long mid = low + (high - low) / 2;

            if (check(mid)) {

                ans = mid;       // 当前 mid 可行，记录答案

                high = mid - 1;  // 尝试挑战更小的极限风险值

            } else {

                low = mid + 1;   // 当前 mid 不可行，说明条件太苛刻，需放宽风险值

            }

        }

  

        cout << ans << "\n";

    }

  

    return 0;

}
```
